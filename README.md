# BioAssay Studio v2.8.1

一个无需服务器、可离线运行的 qPCR、Western blot、蛋白定量与实验知识库浏览器工具。

- qPCR：CSV/Excel 导入、工作表与列映射、Ct 质控、内参与对照组设置、ΔΔCt、逐基因扩增效率校正、生物学重复汇总、误差线图和结果导出。
- WB 单图定量：任意泳道数及每泳道多条带的自动候选；自动模式会按稳健主条带行排除偏离明显的边缘曝光伪影，填写期望泳道数时则以实验约束优先；自动不准时可一键清除自动框后重新手画，也可保留自动框并直接拖动、使用四边/四角手柄缩放；另支持逐条删除、数值编辑、撤销/重做、四种背景扣除、纵向灰度曲线、峰值吸附、SNR/饱和度质控、归一化与 CSV 导出。
- 目的蛋白/内参双图：两图独立框选或复制 ROI、按泳道配对、目的/内参归一化、相对表达量，以及按内参灰度反推建议上样体积。
- WB 图注与排版：默认按全部保留条带的真实边界自动紧凑裁剪、完整居中并仅保留少量左右膜背景，黑框外空白由原始膜边缘背景自然填充；删除或增加黄线后会立即重新计算裁剪、大小和位置，再提供等比缩放、独立上下位置及黑框尺寸作为手动兜底；支持水平校正、蛋白名/分子量/泳道名/归一化数值及分组排版、Times New Roman、真实 pt 字号调节和 PNG/SVG/PDF 输出。
- 考马斯亮蓝蛋白定量：按王浩版实验文档预置 0–1.0 mg/mL 标准点、0.1 mL 加样与 5 mL G-250；支持无重复或 3 次重复、手动录入或 Excel/WPS 整表粘贴、空白扣除、可选过原点回归、R²/SD/CV 建议、1–100× 稀释以及测定液浓度、原液浓度和最终 mg/g 自动换算与 CSV 导出。
- 实验配方与实验方法：使用 IndexedDB 建立完全离线的 Recipe、Protocol、Stock、Chemical、OCR_Record、CalculationHistory 与 Attachment 数据库；支持树状目录、标签/收藏、实时自动保存、版本快照、附件、全文搜索、JSON 完整备份与系统深浅色主题。打印中心可单选或组合选择配方、实验方法和母液，设置 A4/A5、横纵方向与字号，自动生成分页科研文档并调用系统打印机或另存为 PDF。
- 智能配方计算器：支持摩尔浓度称量、Stock 稀释、C1V1=C2V2 任意未知数、w/v、v/v 与质量浓度换算、多组分 Buffer、常用试剂分子量、自动母液匹配、标准配制步骤生成、计算历史，以及一键把计算结果永久保存为可再次编辑和重新计算的配方。
- OCR 导入：可离线保存图片、截图、PDF 和文字记录，在浏览器提供本地 TextDetector 时可尝试直接识别；所有识别文字都必须人工确认。没有本地 OCR 能力的浏览器仍支持原文件保存、粘贴/手动校对、浓度单位提示和一键生成配方或 Protocol 草稿，不会调用云端服务。
- 原始数据与项目：TIFF 8/16-bit 原始像素定量（支持的像素布局）、SHA-256/审计报告，以及包含图像、ROI、计算参数和结果的项目 JSON 恢复。

## 使用

在线版直接打开 GitHub Pages 网站即可。离线网页包解压后可双击 `index.html`；若所在浏览器限制 `file://` 页面的文件选择、打印或本地数据库能力，建议使用 Windows 安装版。四个 WB 图片入口使用用户直接点击的原生文件控件，单图画布还支持把图片直接拖入。工作中可使用页面右上角“保存项目”，避免刷新或关闭后丢失。

Windows 安装版使用与网页端完全相同的分析代码，并自带固定版本的 Chromium 运行内核；适合不希望配置浏览器、本地服务器或网络环境的实验室电脑。桌面版仍然在本机处理文件，知识库使用稳定的应用域名保存在本机用户数据目录，卸载程序默认不会删除这些数据。重要记录仍应定期使用知识库中的“导出备份”另存。桌面版只有在用户点击“检查更新”时才连接 GitHub Releases；发现新版本后由用户确认下载，下载完成后可立即重启安装或退出软件时安装。

v2.8.1 新增知识库选择式打印与打印预览，并为 Windows 安装版加入用户主动触发的在线更新流程。v2.8.0 将 Excel、TIFF 和实验知识库代码改为按需载入；WB 排版的候选检测、裁剪与背景估计加入缓存，拖动黄线或修改字号时不再反复扫描整张原图。知识库附件不再在启动时全部读入内存，自动保存按记录独立排队，备份导入使用完整校验和事务写入。

## 在线网站版

本目录也是可直接托管的静态网站，不需要数据库或后端计算。上线时仅发布 `index.html`、应用脚本/样式、本地 `vendor` 依赖、隐私页面、Web App Manifest、图标、Service Worker 和许可文件；不要发布 `qa`、`tests`、启动脚本或任何实验数据。

- 在线网页只把程序文件下载到访问者浏览器；WB 图片、Excel、ROI 和结果不通过应用代码上传。
- Service Worker 会缓存程序外壳，首次成功访问后可在断网状态下继续打开；缓存不包含用户导入的实验文件。
- qPCR、WB、项目恢复等全部文件入口都使用直接原生文件控件。
- 网页离线压缩包可在本仓库 GitHub Releases 中下载。
- Windows 安装包可按 [`desktop/README.md`](desktop/README.md) 构建；正式 Release 同时提供安装包、`latest.yml` 与 blockmap。
- GitHub Pages 自动发布配置位于 `.github/workflows/deploy-pages.yml`。

公开部署前应阅读 `privacy.html` 与 `THIRD_PARTY_NOTICES.txt`。当前SheetJS已升级至0.20.3，替代存在已知文件解析安全问题的旧0.18.5版本。

qPCR 数据至少应包含三列：样本名、基因/靶标名和 Ct 或 Cq 值。支持仪器导出的附加列，并可在第二步手动映射。

## 计算说明

- `ΔCt = Mean Ct(target) − Mean Ct(reference)`
- `ΔΔCt = ΔCt(sample) − mean ΔCt(control)`
- `Fold change = 2^(−ΔΔCt)`
- 效率校正结果使用每个基因的扩增因子 `E = 1 + efficiency%/100`，先计算 `E_target^(−Ct_target)` 与多个内参 `E_ref^(−Ct_ref)` 的几何平均之比，再相对于对照组归一化。没有标准曲线时保留 100%。
- 生物学重复汇总表给出均值、SD 和 SEM；只有映射了独立的生物学重复列时，这些量才代表独立重复。
- qPCR 结果图的误差线使用当前数据中的 FC SD（由 Ct SD 与 ΔΔCt 误差传播估算），不是置信区间，也不能替代基于生物学重复的统计检验。
- WB 定量采用 ImageJ 同类口径：`IntDen = ROI 平均灰度 × ROI 面积`，并同时显示/导出 Area、Mean、Min、Max、原始 IntDen 和背景校正 IntDen；有效强度可反转，使深色条带为高信号。
- WB 可选择所有背景 ROI 平均、最近背景 ROI、条带左右局部背景或二维稳健背景平面；归一化需要从条带 ROI 中选定一个参照条带。
- 双图建议上样体积 = 当前上样体积 × 基准泳道内参灰度 ÷ 当前泳道内参灰度。仅在线性曝光、内参未饱和且样品浓度可比时有意义。
- 考马斯亮蓝测定液浓度 `C = (扣空白后的 A595 − 回归截距) ÷ 回归斜率`；原液浓度 = `C × 稀释倍数`；最终蛋白含量 (mg/g) = `原液浓度 (mg/mL) × 提取总体积 (mL) ÷ 样品质量 (g)`。R² 阈值只提供实验质量建议，不会自动删除标准点或阻止计算。
- 固体摩尔配制使用 `m (g) = C (mol/L) × V (L) × MW (g/mol)`；母液稀释使用 `C1V1 = C2V2`；w/v 按每 100 mL 中溶质克数计算，v/v 按终体积比例计算。v/v 与质量浓度之间没有密度时不会强行互换。

## 实验知识库

第五模块默认采用桌面浏览器三栏布局：左侧为五个子页面与配方/方法/母液树状目录，中间为编辑器、计算器或搜索结果，右侧为分类、标签、收藏、版本和附件属性。所有表单实时写入 IndexedDB，无需点击保存。

多组分计算时，“自动选择”会先查找同名母液；若找到相容的摩尔浓度母液，则按 C1V1=C2V2 计算加入体积，否则使用试剂分子量计算称量质量。修改母液浓度后，再次打开或重新计算引用该母液的配方即可获得更新结果。每次计算都会保存输入、结果、依据和时间；“保存到实验配方”会把实际加入量与自动步骤一起永久保存。

知识库与页面右上角的 qPCR/WB 项目 JSON 相互独立。请使用知识库工具栏中的“导出备份”生成包含附件和 OCR 原文件的 JSON 备份；清除浏览器站点数据会删除 IndexedDB 知识库。

首版不内置大型 AI 模型，也不把图片发送到在线 OCR。自然语言 AI 配方助手和更大的离线 OCR 语言模型仅保留接口，待能够在离线体积、准确率与许可证之间取得可靠平衡后再启用。

## WB 自动识别

“自动识别条带与背景”采用非 AI 的可解释流程：先定位并裁去无关膜边，再按多条带中心线估计倾斜角，相当于先做一层不改动原始定量像素的虚拟 Straighten；随后从横向灰度面积投影寻找连续泳道区域，并在每个泳道内用去基线的一维灰度曲线定位条带。一个宽条带内部即使出现两个局部峰，只要二者之间没有形成足够宽的背景低谷，就仍作为同一 ROI 进行面积积分，不会拆成两个泳道。未填写“每泳道条带数”时只取每泳道主条带，避免把背景纹理或重影误标为 B2；真正的上下多排图应明确填写 2、3 等数量。泳道数量不是固定值；已知数量时填写“预期泳道数”（不含 Marker），算法只会在明显的双倍大间距中央寻找真实的弱局部面积峰，不按整图等距硬切，也不会从相邻宽带尾部任意补峰。v2.8.0 会根据每条带的真实左右信号边缘紧凑收框，只有信号真正触及搜索边界时才单侧增加保护距离；“条带左右留白（px）”可在 0–30 px 间调节，默认 4 px。相邻泳道仍进行最终去重，并可用“忽略左侧 Marker 区域”排除标尺或 Marker。自动背景会避开条带及其安全边距。自动框以虚线与“自动”标签显示，可移动、缩放、删除、改名，也可在纵向灰度曲线上点击峰值进行吸附；手动框选永远保留。

每个条带同时显示 SNR、饱和像素比例和质控状态。软件会提示低信噪比、边缘 ROI、横向边界不确定、接近相邻泳道、缺少背景或过曝，但不会凭算法恢复已经饱和的灰度信息。自动结果必须在导出前人工确认。

图注与排版中的手动校正不是只移动中心线：开启后可选择/拖动黄线，也可进入“增加黄线”模式后在黑框内点击新增；选中黄线可显式删除。新增或删除时，软件会恢复 100% 自动紧凑排版，按当前保留集合重新裁剪并居中，同时在相同位置同步修改泳道名称和归一化数值，仍可在文本框中继续手填。名称和数值的条目数只负责标注，永远不会反向限制自动识别数量；只有用户明确填写的“预期泳道/条带数”才会约束识别，留空即自动判断。选择“每张图片分别设置”后，每张图拥有独立的可选期望数；上传新图会清除旧图留下的数量约束。黄线、泳道名称和归一化数值共用同一个泳道中心坐标；字号变化时，软件依据实际文字宽度和相邻泳道间距自适应旋转角度，所有名称在靠近黑框的一端保持统一基线并只向框外延伸，PNG 与 SVG 使用相同布局。

自动识别适合边缘裁干净、条带横向且未过曝的图像。对弯曲泳道、严重拖尾、饱和曝光、多张膜拼图或带有文字/标尺的图片，请先裁图并逐项复核候选框后再导出。

## 目的蛋白 / 内参双图

在 WB 模块顶部切换到“目的蛋白 / 内参双图”。分别上传内参和目的蛋白图片后，可以在任一侧手动拖动框选，或使用自动框选；两图泳道顺序一致时，可将一侧的 ROI 布局按图像尺寸比例复制到另一侧。工具以泳道编号配对，采用可开关的局部背景扣除，计算：

- `目的/内参 = 目的蛋白背景扣除积分灰度 / 内参背景扣除积分灰度`
- `相对表达量 = 当前泳道的目的/内参 / 基准泳道的目的/内参`

“自动框选”和“复制 ROI”均为候选辅助，不能替代对齐、条带身份和曝光饱和度的人工复核。

## 600 DPI 图像导出

排版工作区默认按 180 mm、600 DPI 生成 4252 px 宽 PNG，也可选择 85 mm 单栏或自定义物理宽度。蛋白名、kDa、归一化数值和泳道名称的字号统一使用真实 pt；软件按当前物理宽度换算预览画布，导出 DPI 改变时印刷字号不变。PNG 中会写入标准 `pHYs` 物理分辨率元数据；界面在导出前显示实际毫米、像素和 DPI。SVG 仍保持可编辑，PDF 通过打印流程输出。

## 本地 WB 回归语料

本地 WB 图像语料库仅作为只读、非 AI 的确定性回归数据使用。发布包不包含、上传或复制任何原始实验图片；由于归档图片尚无逐图人工真值标注，数量分布只用于发现回归，不宣称为识别准确率。

## 非 AI 设计参考

- [ImageJ](https://github.com/imagej/ImageJ) 的 ROI、等尺寸泳道、泳道灰度曲线、峰面积积分、背景扣除和人工复核工作流。
- ImageJ `GelAnalyzer` 的“二维泳道压缩为一维 profile，再测峰面积”思路；本工具将同一思想扩展为自动候选、低信号补全和浏览器内手动纠错。
- [GelAnalyzer](https://gelanalyzer.com/docs/baseline/) 的 rolling-ball / morphological / valley-to-valley 基线与可编辑控制点设计。当前浏览器版本已采用局部曲线去基线和二维背景平面；可编辑 profile 基线仍属于后续增强项。
- [GelJ](https://pmc.ncbi.nlm.nih.gov/articles/PMC4549892/) 的横向投影定位泳道、逐泳道纵向局部峰、全局/逐泳道阈值和手动修正组合。
- [RQdeltaCT](https://github.com/Donadelnal/RQdeltaCT) 的 ΔCt/ΔΔCt、质控和结果导出流程。
- [deskew](https://github.com/sbrunner/deskew) 的角度检测后反向旋转思路，用于排版工作区的水平校正。

本项目没有引入 AI 模型，也未直接复制上述项目源码；采用的是公开、可解释的分析工作流，并保留人工复核作为最终决定。

这是用于数据整理与可视化的轻量工具。显著性检验应根据生物学重复、分布及实验设计在专门的统计流程中完成。
